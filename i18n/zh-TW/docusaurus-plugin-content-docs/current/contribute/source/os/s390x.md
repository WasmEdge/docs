# 在 IBM LinuxONE(s390x) 上建置

請依照本教學從原始碼在 s390x 硬體上建置並測試 WasmEdge。

## 建置 WasmEdge

### 取得原始碼

```bash
git clone https://github.com/WasmEdge/WasmEdge.git
```

### 相依套件

WasmEdge 至少需要 LLVM 18,您可能需要自行安裝下列相依套件。

:::note
由於 Ubuntu 22.04 與較舊版本的 Ubuntu 不包含 llvm-18 套件,您可以改用 LLVM 官方儲存庫來安裝 llvm-18。

```bash
wget https://apt.llvm.org/llvm.sh
chmod +x llvm.sh
sudo ./llvm.sh 18
```
:::

```bash
sudo apt-get update
sudo apt install -y software-properties-common cmake gcc g++
sudo apt install -y llvm-18-dev liblld-18-dev
```

### 編譯

關於所有 CMake 選項的描述,請參閱[此處](../build_from_src.md#cmake-building-options)。

```bash
cd WasmEdge
mkdir -p build && cd build
cmake -DCMAKE_BUILD_TYPE=Release .. && make -j
```

## 執行測試

下列測試僅在建置選項 `WASMEDGE_BUILD_TESTS` 設為 `ON` 時可用。

使用者可使用這些測試來驗證 WasmEdge 執行檔的正確性。

```bash
cd <path/to/wasmedge/build_folder>
LD_LIBRARY_PATH=$(pwd)/lib/api ctest
```

## 建置支援的外掛
目前,s390x 支援 [WASI-NN 外掛](../plugin/wasi_nn#build-with-llamacpp-backend-on-linux),使用 GGML 後端。
建議建置具有 BLAS 支援的外掛。要建置具有 BLAS 支援的外掛,首先安裝 OpenBLAS 函式庫:

```bash
sudo apt install libopenblas-dev
```

接著,使用下列指令建置 WasmEdge:

```bash
cd <path/to/wasmedge/build_folder>
make -DCMAKE_BUILD_TYPE=Release -DWASMEDGE_BUILD_PLUGINS=ON -DWASMEDGE_PLUGIN_WASI_NN_BACKEND="GGML" -DWASMEDGE_PLUGIN_WASI_NN_GGML_LLAMA_BLAS=ON ..
```

關於將模型轉換為 big endian 的指南,請參閱 https://github.com/ggml-org/llama.cpp/blob/master/docs/build-s390x.md#getting-gguf-models
