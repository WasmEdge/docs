## 文件風格指南

### 語言

- 使用清晰、簡單、簡潔的語言。盡可能避免行話與技術術語。如果無法避免,請提供清楚的定義或說明。
- 採用主動語態書寫,並使用第二人稱(「您」),讓文件更加以使用者為中心。

### 結構與格式

- 使用具描述性的標題與子標題來建構內容結構。
- 透過項目符號與編號清單讓內容更易於閱讀與遵循。
- 必要時包含程式碼範例與技術參考資料。它們應格式良好且易於理解。對程式碼範例請使用程式碼區塊與語法標示。

### 內容

- 以提供主題概觀的引言作為開頭。
- 提供逐步指示,並在必要時包含程式碼範例。
- 包含疑難排解章節,協助使用者解決可能遇到的常見問題。

### 其他考量

- 鼓勵社群貢獻。包含一節說明使用者如何為專案貢獻。
- 定期檢視內容並在必要時進行更新。
- 確保文件對所有人皆可存取。應該易於閱讀、理解與導覽。

## 程式碼風格指南

### 程式碼格式

- 使用一致的縮排。例如,您可以選擇使用空格而非 tab,並在整個專案中保持一致。
- 使用有意義的變數、函式與類別命名。它們應清楚表示變數的內容、函式的功能等。
- 為程式碼加上註解。說明每個區段或每一行程式碼的功能,尤其是當其涉及複雜邏輯時。

### 程式碼品質

- 保持程式碼 DRY (Don't Repeat Yourself,不要重複自己)。如果您發現自己在多個地方撰寫相同的程式碼,請考慮建立函式或類別。
- 撰寫小型、單一目的的函式。每個函式應只做一件事,並且做好。
- 妥善處理錯誤。不要在程式碼中留下空的 catch 區塊。

### 測試

- 為程式碼撰寫測試。這有助於及早發現臭蟲,並確保程式碼如預期般運作。
- 遵循一套測試方法,例如單元測試或整合測試。

### 安全性

- 避免可能導致安全性漏洞的程式碼,例如 SQL injection。
- 使用安全的函式與函式庫。
- 遵循 CNCF 提供的安全性最佳實務。
